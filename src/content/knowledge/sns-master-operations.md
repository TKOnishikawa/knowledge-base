## 1. 週間スケジュール

### Week A: 記事公開週

| 曜日 | やること | 時間 |
|------|---------|------|
| **月曜** | X投稿 A（Practice: 実践記録） | 20分 |
| **水曜** | X投稿 B（Insight: 知見共有）+ 記事リンク | 20分 |
| **金曜** | X投稿 C（Candid: 本音・裏話） | 20分 |
| **土曜** | YouTube動画 公開 | 1.5時間 |
| **随時** | HP記事 執筆 | 3時間 |
| | **合計** | **約6.5時間** |

### Week B: 記事なし週

| 曜日 | やること | 時間 |
|------|---------|------|
| **月曜** | X投稿 A（Practice: 実践記録） | 20分 |
| **水曜** | X投稿 B（Insight: 知見共有） | 20分 |
| **金曜** | X投稿 C（Candid: 本音・裏話） | 20分 |
| **土曜** | YouTube動画 公開 | 1.5時間 |
| **日曜** | 週次振り返り | 30分 |
| | **合計** | **約3.5時間** |

**月平均: 週5時間**（6.5h + 3.5h ÷ 2 = 5.0h）

### 時間予算の内訳

| カテゴリ | 週あたり | 割合 |
|---------|---------|------|
| X投稿 ×3 | 1時間 | 20% |
| YouTube ×1 | 1.5時間 | 30% |
| HP記事（月平均） | 1.5時間 | 30% |
| エンゲージメント（リプ・DM対応） | 0.5時間 | 10% |
| 振り返り | 0.5時間 | 10% |
| **合計** | **5時間** | **100%** |

> **鉄則**: 週5時間を超えたら「やりすぎ」。品質を下げてでも時間内に収める。完璧な投稿1本より、まあまあの投稿3本の方が価値がある。

## 2. 制作フロー

### HP記事 → 派生コンテンツの変換フロー

```
[HP記事（原液）]
    │
    ├─→ [YouTube動画]
    │     HP記事 → Claude（台本化） → ElevenLabs（音声） → Canva/Gamma（スライド） → CapCut（字幕・編集）
    │     所要時間: 約1.5時間
    │
    └─→ [X投稿 ×3]
          HP記事の要点を3テンプレートに分解
          Practice: 記事で扱った実践を「今日やった」形式に
          Insight: 記事の核心を「よくある失敗」形式に
          Candid: 記事の裏話・執筆動機を「正直に言うと」形式に
          所要時間: 各20分
```

### 各チャネルの制作ツールチェーン

#### X投稿（20分/本）

1. テンプレート（A/B/C）を選択
2. Claude にドラフト作成を依頼（型 + 素材を入力）
3. 自分の体験・数字を差し込み
4. 言い回し・温度感を最終調整
5. 投稿

#### YouTube動画（1.5時間/本）

| ステップ | ツール | 時間 |
|---------|-------|------|
| 台本作成 | HP記事 → Claude | 0分（記事が原液） |
| 音声生成 | ElevenLabs | 10分 |
| スライド作成 | Canva or Gamma | 40分 |
| 字幕・編集 | CapCut | 30分 |
| 概要欄・タグ | Claude → 手動調整 | 10分 |

**YouTube制作の鉄則**:
- 顔出し不要。スライド + AI音声で十分
- 1本30分が理想だが、初期は1.5時間かかっても OK
- クオリティの70%はHP記事（原液）の質で決まる

#### HP記事（3時間/本、隔週）

1. テーマ決定（月次レビューで翌月分を決定）
2. リサーチ（Claude / Grok API / 実務経験から）
3. 構成設計（H2 5-7本）
4. 執筆（Claude ドラフト → 手動編集）
5. SEOチェック（タイトル・メタ・H2にキーワード配置）
6. GitHub Pages にデプロイ

## 3. 投稿前チェックリスト

投稿ボタンを押す前に、以下6項目を確認する。

- [ ] **ペルソナ**: この投稿は、プライマリ or セカンダリペルソナに刺さるか？（→ [Strategy: ペルソナ](/knowledge-base/knowledge/sns-master-strategy/)）
- [ ] **トーン**: トーン・オブ・ボイスに合っているか？上から目線になっていないか？（→ [Creative: トーン](/knowledge-base/knowledge/sns-master-creative/)）
- [ ] **NG確認**: NG事項に抵触していないか？（クライアント名、煽り文句、攻撃的表現）
- [ ] **機密情報**: APIキー、パスワード、クライアント固有情報が含まれていないか？
- [ ] **CTA/導線**: 次の行動への導線があるか？（HP記事リンク、概要欄リンク、フォロー促進）
- [ ] **タグ/キーワード**: ハッシュタグ・キーワードが適切か？（過剰でもなく、不足でもなく）

> **80点ルール**: チェックリスト6項目中5項目OKなら投稿してよい。完璧を目指して投稿しないことの方が損失。

## 4. KPI設計

### コントローラブル指標（主KPI）

**自分で制御できる指標。これを毎週追う。**

| 指標 | 目標 | 計測方法 |
|------|------|---------|
| X投稿数 | 週3本 | SNSポータルで確認 |
| YouTube公開数 | 週1本 | SNSポータルで確認 |
| HP記事公開数 | 隔週1本 | SNSポータルで確認 |
| 制作時間 | 週5時間以内 | SNSポータルの `timeSpentMinutes` を集計 |
| チェックリスト通過率 | 100% | 投稿前に毎回確認 |

### アウトカム指標（参考）

**結果として出る指標。月次で確認。一喜一憂しない。**

| 指標 | 計測方法 | 備考 |
|------|---------|------|
| X インプレッション数 | X Analytics | 月間合計 |
| X エンゲージメント率 | X Analytics | いいね + RT + リプ / インプレッション |
| YouTube 視聴回数 | YouTube Studio | 月間合計 |
| YouTube 平均視聴時間 | YouTube Studio | 動画ごとに確認 |
| YouTube 登録者数 | YouTube Studio | 純増数 |
| HP PV | Google Analytics | 月間合計 |
| 問い合わせ数 | フォーム + DM | 月間合計 |

### 3ヶ月マイルストーン（現実ライン）

| 指標 | 目標 | 根拠 |
|------|------|------|
| Xフォロワー | 100〜300 | 新規アカウント + 週3投稿の現実的な伸び |
| YouTube登録 | 30〜100 | スライド動画 + 新規チャンネルの現実ライン |
| 月間問い合わせ | 1〜3件 | 初期はほぼゼロ。3ヶ月目に1件あれば上出来 |
| コンテンツ資産 | 計40本以上 | X 36本 + YouTube 12本 + HP記事 6本 |

> **注意**: v2戦略では「Xフォロワー500-1000、YouTube登録100-300」と記載しているが、1人運用の新規アカウントではこれは楽観的。上記が現実的な目標。達成したら上方修正する。

### KPI運用の原則

1. **コントローラブル指標が主、アウトカム指標は従**。投稿数は自分で制御できるが、フォロワー数は制御できない
2. **アウトカム指標は月次で確認**。毎日見ると一喜一憂して疲弊する
3. **数字が伸びなくても、投稿を続けること自体が価値**。3ヶ月のコンテンツ蓄積は「資産」
4. **3ヶ月で一度も振り返りをしなかったら、それが最大の失敗**

## 5. 振り返りルール

### 週次振り返り（日曜 30分）

**やること**:
1. SNSポータル（`/sns/`）を開く
2. 今週の投稿を確認（計画通り投稿できたか？）
3. 各投稿のメトリクスを frontmatter に記録（`metrics.impressions`, `metrics.engagements`）
4. `learnings` フィールドに気づきを1行書く
5. 翌週のX投稿テーマを3つ決める

**所要時間**: 30分。超えたら打ち切る。

### 月次レビュー（月末 1時間）

**やること**:
1. アウトカム指標の確認（フォロワー推移、視聴回数、PV、問い合わせ）
2. コントローラブル指標の達成率確認（投稿数、制作時間）
3. 反応が良かった投稿 TOP3 の分析（なぜ刺さったか？）
4. 反応が悪かった投稿の分析（何が足りなかったか？）
5. 翌月のHP記事テーマを2本決める
6. 翌月のYouTubeテーマを4本決める

**アウトプット**: 月次ログを SNSポータルに `log/2026-mMM.md` として記録。

### 四半期レビュー（3ヶ月ごと 2時間）

**やること**:
1. 3ヶ月マイルストーンの達成度確認
2. ペルソナの妥当性再検証（実際に反応しているのは想定ターゲットか？）
3. チャネル戦略の見直し（追加すべきチャネル、撤退すべきチャネル）
4. KPI目標の上方/下方修正
5. **このマスタードキュメント自体の更新**
