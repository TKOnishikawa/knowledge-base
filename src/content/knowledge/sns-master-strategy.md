## 1. ミッション

> **事業推進 = コンテンツ生成。**
> 「発信のために仕事を止める」のではなく、「仕事の記録が発信になる」設計にする。

### なぜSNSをやるか（1文）

中小企業の経営者に「この人に相談したい」と思ってもらうための**認知→信頼→接触の最短ルート**を構築する。

### SNSの事業上の価値

- **認知**: 広告費ゼロで専門性を届けられる唯一のチャネル
- **信頼**: 「実際にやったこと」の蓄積が、営業トーク以上の説得力になる
- **資産化**: YouTube・HP記事は検索流入で長期的にリードを生む（フロー→ストック変換）
- **差別化**: 「コーチング×コンサル×エンジニアリング」の統合を体現する場

### SNSをやらない場合の機会損失

- 紹介依存から抜け出せない（月0-1件の紹介頼み）
- 専門性の証明手段がない（「何ができる人か」が外から見えない）
- 価格競争に巻き込まれる（HPだけだと「安さ」で比較される）

## 2. ブランドアイデンティティ

### 名称ルール

| 用途 | 名称 | 備考 |
|------|------|------|
| 法人名 | **RAFTEL Inc.** | 正式な法人名称 |
| サービスブランド | **omoshiku** | SNSアカウント名・HPドメイン・対外ブランド |
| Xアカウント | @nishikawa_omoshiku | 実名 + ブランド名 |
| YouTubeチャンネル | omoshiku | ブランド名で統一 |

**判断基準**: 対外的な発信では「omoshiku」ブランドを使用。法人契約・請求書等の公式文書では「RAFTEL Inc.」を使用。

### タグライン

> **想いを、仕組みに。**

### 創業者ストーリー（1パラグラフ）

マクロミルでデータ分析、MTGでコーチング、アクセンチュアでDXコンサルティング。3社の経験を統合し、中小企業が「やりたいこと」を「回る仕組み」に変えるサービスを提供。コーチングで想いを引き出し、コンサルで戦略を設計し、エンジニアリングで実装する——この一気通貫が強み。

### ブランドパーソナリティ（5形容詞）

1. **実践的** — 理論だけの人と差別化。「実際にやった」ベースで語る
2. **誠実** — 失敗談も隠さない。できないことは「できない」と言う
3. **対話的** — 「教える」より「一緒に考える」。上から目線にならない
4. **専門性のある** — データ・AI・Webの技術的裏付けがある
5. **温かい** — コーチングマインド。相手の可能性を信じる

## 3. ターゲットペルソナ

### 3a. プライマリ: 中小製造業の経営者

| 属性 | 詳細 |
|------|------|
| 業種 | 精密部品加工（切削・研磨・板金等） |
| 売上規模 | 5〜20億円 |
| 従業員数 | 10〜40名 |
| IT担当 | 不在（社長 or 総務が兼任） |
| 世代 | 2代目〜3代目が多い（40-55歳） |

**本音の痛みランキング**（建前ではなく実際の優先順位）:

1. **人材確保** — 若手が来ない、ベテラン退職で技術喪失
2. **原価が見えない** — Excelと感覚頼み。「本当に儲かってるのか」分からない
3. **見積が社長しかできない** — 属人化。社長不在で業務停止
4. **FAX/Excel脱却** — 分かってるけど変え方が分からない
5. **顧客集中リスク** — 売上の50-80%が大手1-3社依存
6. **HP放置** — 「やらなきゃとは思うが後回し」
7. **SNS** — 「うちには関係ない」

**コンテンツ企画への示唆**: HPやSNSはペインランキング6-7番目。直球で「HP作りましょう」ではなく、**1-5番目の痛みに共感してから「その解決手段としてのHP/データ基盤」**という文脈で届ける。

**情報接触チャネル**: YouTube検索（「○○ 作り方」型）、業界団体の勉強会、同業の知人紹介、展示会

> 詳細なペルソナ分析は [中小製造業ターゲットペルソナ分析](/knowledge-base/knowledge/sme-manufacturing-target-persona/) を参照。

### 3b. セカンダリ: 非製造業の中小企業経営者

| 属性 | 詳細 |
|------|------|
| 業種 | サービス業、小売、士業、不動産等 |
| 売上規模 | 1〜20億円 |
| 従業員数 | 5〜50名 |
| IT担当 | 不在 |
| 共通の痛み | 「何から手を付けていいか分からない」 |

**プライマリとの違い**: 製造業ほどFAX依存・原価管理の深刻さはないが、「DXの入口が見えない」「HP制作で失敗した経験がある」という共通項がある。

**コンテンツでの使い分け**: YouTube Ep.1（HP Before/After）のような「入口コンテンツ」はセカンダリにもリーチする。原価管理・見積属人化の深いコンテンツはプライマリ特化。

### 3c. アンチペルソナ（ターゲットにしない層）

| 層 | 理由 |
|----|------|
| 大企業のIT部門 | 予算規模・意思決定プロセス・調達ルールが合わない |
| 個人事業主・フリーランス | サービス単価と合わない（¥50K〜の価格帯に見合わない） |
| 「とにかく安くHP作りたい」層 | 価格だけで判断する層とは価値観が合わない |
| テック企業・スタートアップ | 自社にエンジニアがいる。外部支援の必要性が薄い |

## 4. チャネル戦略（概要）

| チャネル | 役割 | 頻度 | 週あたり時間 |
|---------|------|------|------------|
| **X** | 認知拡大 × 信頼構築 | 週3（月・水・金） | 1h |
| **YouTube** | 長期資産化 × 信頼の深掘り | 週1（土曜公開） | 1.5h |
| **HP記事** | SEO基盤 × 全コンテンツの源泉 | 隔週1本 | 1.5h（月平均） |

**合計: 週5時間**（記事あり週: 約6.5h / 記事なし週: 約3.5h）

### コンテンツの流れ

```
HP記事（原液） → YouTube（動画化） → X投稿3本（派生）
```

HP記事が全コンテンツの「源泉」。1つの記事から YouTube1本 + X投稿3本を派生させることで、1コンテンツあたりの効率を最大化する。

### Phase 2 候補（現時点ではやらない）

- **LinkedIn**: B2Bリード獲得率は最高水準だが、フォロワー基盤が必要。6ヶ月後に検討
- **リードマグネットPDF**: 問い合わせが月10件超えたら検討
- **メール自動化**: リードマグネットの次段階

> チャネル評価の詳細（カットリスト含む）は [SNS戦略 v2](https://tkonishikawa.github.io/raftel-hp/content/sns-strategy-v2.html) を参照。

## 5. ファネル設計

### 3ステップファネル

| ステップ | 感情 | チャネル | 判定基準 |
|---------|------|---------|---------|
| **Step 1: 認知** | 「この人、面白い視点持ってるな」 | X / YouTube | フォロー・チャンネル登録 |
| **Step 2: 信頼** | 「この人、ちゃんと実績あるんだ」 | HP記事 | 記事を複数回閲覧 / 滞在時間 |
| **Step 3: 接触** | 「30分、話を聞いてもらえませんか」 | DM / 問い合わせフォーム | 問い合わせ送信 |

### 各ステップの設計ポイント

- **Step 1 → 2**: プロフィールにHP記事へのリンク。X投稿・YouTube概要欄からHP記事への導線を必ず設置
- **Step 2 → 3**: 記事末尾に明確なCTA（「30分無料相談」等）。問い合わせのハードルを下げる
- **Step 3 → 商談**: 30分の無料相談で課題をヒアリング。Quick Launch（¥50K）or Growth Partnerの提案

### Phase 2 で追加検討するもの

- リードマグネットPDF（メアド取得→ナーチャリング）
- メール自動化（ステップメール）
- LINE公式アカウント

## 6. 競合・ポジショニング

### 直接競合

| 競合タイプ | 例 | 強み | 弱み |
|-----------|----|----|------|
| 中小企業向けHP制作会社 | 地域のWeb制作会社 | 実績多数、対面サポート | 戦略設計が弱い。「作って終わり」が多い |
| DXコンサルティング会社 | 大手SIer子会社 | 組織力、実績 | 単価が高い（月額50万〜）。中小に合わない |
| YouTube「HP制作」系チャンネル | 個人YouTuber | 情報量、SEO | 実行支援がない。見ても「自分ではできない」 |

### 差別化ポイント

**「コーチング × コンサル × エンジニアリング」を1人で統合できる人は市場にほぼいない。**

- コーチング: 経営者の「想い」を引き出す（多くのITベンダーにはこの力がない）
- コンサル: 戦略を設計する（Web制作会社には「上流」の経験がない）
- エンジニアリング: 自分で実装する（コンサルは「提案して終わり」が多い）

### ポジショニング（2軸マップ）

```
技術深度 高
    │
    │  ● omoshiku
    │    （コーチング×コンサル×エンジニアリング統合）
    │
    │          ○ DXコンサル
    │            （高単価・大企業向け）
    │
    │  ○ Web制作会社
    │    （作るだけ）
    │
    │              ○ YouTuber
    │                （情報だけ）
    │
    └────────────────────── 伴走度 高
```

omoshikuは「技術も深く、伴走もする」という右上のポジション。このポジションを取れている競合はほぼ存在しない。
