## 1. トーン・オブ・ボイス

### 基本トーン

**「専門家だが偉そうでない、実践ベースで語る、本音を隠さない、一緒に考えるスタンス」**

- 理論だけの評論家にならない。「実際にやった」ことベースで語る
- 失敗談も出す。完璧な人ではなく、「一歩先を歩いてる先輩」のイメージ
- 「教えてあげる」ではなく「一緒に考えましょう」のスタンス
- 専門用語は使ってもいいが、必ず平易な言い換えを添える

### トーンの4軸（スライダー）

```
フォーマル  ○ ○ ● ○ ○  カジュアル    → やや カジュアル寄り
専門的     ○ ● ○ ○ ○  平易         → 専門寄りだが難語は避ける
真面目     ○ ○ ● ○ ○  ユーモア     → バランス型（たまに軽口OK）
控えめ     ○ ○ ○ ● ○  自信あり     → 実績ベースの落ち着いた自信
```

### チャネル別トーン調整

| チャネル | トーン特性 | 文体 |
|---------|----------|------|
| **X** | 簡潔・断定的・問題提起型 | 体言止め OK。「〜だ。」「〜と思う。」。140字で核心を突く |
| **YouTube** | 落ち着いた語り口・ストーリーテリング型 | 「〜ですよね」「〜なんです」。共感→展開→結論の流れ |
| **HP記事** | 丁寧・論理的・SEO意識した構造 | 「〜です/ます」調。H2/H3で論理構造を明示 |

## 2. 言葉遣いルール

### 使う言葉 / 使わない言葉

| 使う | 使わない | 理由 |
|------|---------|------|
| 仕組み化 | DXトランスフォーメーション | カタカナ過多で空虚 |
| 見える化 | 可視化ソリューション | 営業っぽすぎる |
| 1人で全部やる | ワンストップサービス | 差別化が伝わらない |
| 当たり前を武器に変える | ベストプラクティスの導入 | 抽象的すぎる |
| ±0.01mm, 24時間以内 | 高品質, 迅速な対応 | 数字で具体化すべき |
| 「実はこれ、よくある失敗で…」 | 「御社の課題を解決します」 | 共感が先、提案は後 |
| 〜を一緒に考える | 〜をご提案いたします | 上から目線を避ける |

### 数字の使い方

- **具体的な数字を優先**: 「多くの企業」→「73%の中小製造業」
- **自分の実績も数字で**: 「多数の経験」→「累計15社のHP制作支援」
- **ただし盛らない**: 検証できない数字は使わない。「約」「およそ」で正直に

### 一人称・呼称

| 場面 | 使う表現 |
|------|---------|
| 自分 | 「僕」（Xカジュアル） / 「私」（HP記事・YouTube） |
| 相手 | 「社長」「経営者の方」（B2B） / 「あなた」は使わない |
| 自社 | 「omoshiku」 / 「うち」（カジュアル文脈のみ） |

## 3. コンテンツテンプレート

### X投稿 3テンプレート

#### A: Practice（実践記録）— 月曜

**構造**: [今日やったこと] → [気づき] → [一般化]

**例文**:
> 今日クライアントのBQダッシュボードを構築した。
> 「数字を見る習慣がなかった」という社長が、翌朝から毎日確認するようになった。
> ツールの導入よりも「見たくなる設計」が大事だと改めて実感。

**ポイント**:
- 主語は「今日」「今週」「先日」— 鮮度がある言葉で始める
- 具体的なツール名・技術名を入れる（専門性の証明）
- 最後の1文は「だから何？」に答える一般化

#### B: Insight（知見共有）— 水曜

**構造**: [よくある失敗] → [原因分析] → [解決の方向性]

**例文**:
> 中小企業のHP制作でよくある失敗:
> 「とりあえずWordPress」で始めて、更新が止まる。
> 原因はたいてい「更新するスキルがない」こと。
> ノーコード × AI × テンプレートで「更新の仕組み」まで設計しないと、HPは負債になる。

**ポイント**:
- 冒頭で「あるある」を突く（共感フック）
- 原因を指摘する（表面的な対処法ではなく根本原因）
- 解決策は具体的すぎず、「方向性」レベルで留める（詳細はHP記事に誘導）

#### C: Candid（本音・裏話）— 金曜

**構造**: [正直な告白] → [学んだこと] → [読者への問い]

**例文**:
> 正直に言うと、起業して最初の3ヶ月は売上ゼロだった。
> 「自分の価値を言語化できない」が最大の壁だった。
> コーチングで自分自身が壁にぶつかった経験が、今クライアントの支援に活きてる。

**ポイント**:
- 弱さを見せる（完璧な人間は信頼されない）
- ただし「かわいそう」ではなく「そこから学んだ」で締める
- 読者に問いかけるか、共感を促す終わり方

### AI活用のルール（X投稿）

1. テンプレートの「型」に沿って、Claudeにドラフトを書かせる
2. 自分の体験・数字を差し込む
3. 最後に「人間の温度」を足す（微調整・言い回し変更）
4. **1投稿あたり約20分**が目安

### YouTube動画テンプレート

**標準構成（7-8分）**:

| 時間 | セクション | 目的 |
|------|----------|------|
| 0:00-0:15 | フック | 「見覚えありませんか？」型の問題提起。15秒で興味を引く |
| 0:15-1:00 | 問題提起 | あるある5選。共感を集める |
| 1:00-1:30 | 原因分析 | なぜこうなるのか。構造的な問題を指摘 |
| 1:30-4:00 | 解決策（実演） | Phase 1（戦略設計）の具体的なやり方を見せる |
| 4:00-5:00 | AI活用 | プロンプト→生成結果のダイジェスト |
| 5:00-6:00 | Before/After | クライマックス。劇的な比較で感動を演出 |
| 6:00-6:30 | まとめ | 3つのポイントに整理 |
| 6:30-7:00 | CTA | 次のアクション（概要欄リンク + チャンネル登録） |

**サムネイルルール**:
- **Before/After構図**: 左=暗め（問題）、右=明るめ（解決）
- **テキスト3行以内**: メイン（大）+ サブ1 + サブ2
- **色**: amber背景ベース or ダーク背景 + amber文字
- **フォント**: 太め・視認性優先

### HP記事テンプレート

**標準構成**:
- 文字数: 3,000〜5,000字
- H2: 5〜7本
- 冒頭: サマリ（この記事で分かること 3点）
- 末尾: CTA（「30分無料相談」or 「関連記事」）
- SEOキーワード: タイトル + H2 + 本文冒頭に自然に配置

## 5. NG事項

### コンテンツNG

| NG | 理由 |
|----|------|
| 特定の個人・企業を名指しで批判する | 信頼毀損、炎上リスク |
| クライアント名・具体的な契約金額の開示 | 守秘義務違反 |
| 政治・宗教に関する意見表明 | ビジネスアカウントとして不適切 |
| 「○○するだけで」「誰でも簡単に」等の煽り | 信頼性を損なう。専門家ポジションと矛盾 |
| 未検証の統計データの引用 | ファクトチェックできないデータは使わない |
| 他者のコンテンツの無断転載・パクリ | 著作権侵害、信頼毀損 |

### ビジュアルNG

| NG | 理由 |
|----|------|
| フリー素材の安っぽいビジネスマン写真 | ブランドイメージと合わない |
| 蛍光色やネオン系のカラー | ブランドカラーとの整合性がない |
| 文字が読めないほど小さいサムネイル | モバイルで視認できない |
| ブランドカラー以外のアクセント色の無断使用 | ビジュアルの一貫性を壊す |
| 過度なアニメーション・エフェクト | 情報量の邪魔になる |

### 運用NG

| NG | 理由 |
|----|------|
| 他者の投稿に対する攻撃的なリプライ | ブランドイメージ毀損 |
| フォロワー購入・自動いいね等のグロースハック | 発覚時の信頼毀損が致命的 |
| 投稿後の大幅な修正・削除 | 事前チェックで防ぐ。修正が必要な場合は追加投稿で訂正 |
| 深夜・早朝の投稿 | ビジネスアカウントとして不適切な時間帯。予約投稿を活用 |
| 同じ内容のマルチポスト（X/YouTube/HP全部同文） | チャネル特性に合わせた変換が必要 |
