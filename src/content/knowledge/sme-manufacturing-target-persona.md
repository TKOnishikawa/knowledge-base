## 背景（Background）

omoshikuのYouTube企画（Before/After型HP制作動画）の架空企業として「製造業（精密部品加工）」を選定。
動画制作の前に、業界の実態を深く理解し、リアリティのあるコンテンツを作るためにリサーチを実施。
同時に、omoshikuのターゲット顧客としての製造業の可能性を検証する目的もある。

## 目的（Purpose）

- 中小製造業（5-20億帯）のビジネスモデル・課題・DX状況を網羅的に把握する
- YouTube動画の架空企業設定にリアリティを持たせる
- omoshikuのサービスラインナップとの接点・提案機会を整理する

## 期待アウトカム（Expected Outcome）

- YouTube Ep.1（HP Before/After）の架空企業「丸山製作所」の設定根拠
- Ep.2以降の本丸コンテンツ（原価管理・見積属人化・データPDCA）の企画根拠
- omoshikuの営業ターゲット・二段構え戦略（HP入口→DX本丸）の意思決定資料
- 製造業の「建前 vs 本音」のニーズギャップを踏まえたポジショニング設計

## 1. ビジネスモデル・収益構造

| 要素 | 典型パターン |
|------|------------|
| 収益モデル | 受託加工（下請け）= 売上の70-90%。自社製品は10%未満が大半 |
| 取引構造 | 大手メーカー1-3社に依存（売上の50-80%を占める） |
| 受注経路 | 既存顧客からの電話・FAX → 見積 → 受注。新規はほぼゼロ |
| 粗利 | 30-40%（ただし原価把握がExcel/感覚頼みで「本当の粗利が見えてない」） |
| 価格決定権 | なし。「お客さんの言い値」で受ける構造 |
| 参入障壁 | 技術力（精度±0.01mm等）だが、HPに書いてないので外からは見えない |

**核心の問題**: 「技術力はあるのに、それが外に伝わっていない」

## 2. 現状の業務プロセスと痛み

```
[受注] FAX(73%) → 手入力(60%) → Excel台帳
         ↓ 88%が「やめたい」けど取引先がFAX
[見積] 社長の頭の中 → 手計算 → FAX返信
         ↓ 属人化の極み。社長不在=見積不能
[生産] 紙の工程表 → 口頭指示 → ホワイトボード管理
         ↓ 「今どの工程にいるか」事務所からは見えない
[納品] 紙伝票 → 手書き検品 → FAX納品書
         ↓ 月8,000-10,000枚の紙帳票（実例）
[請求] Excelで手入力 → 会計事務所に3ヶ月後に試算表
         ↓ 月次の売上すら正確に把握できない
```

### 統計データ（AI inside調査）
- **95.2%が紙書類をやり取り**、うち73.3%がFAXで受領
- 月平均1,000枚以上のFAXを処理する企業が43.0%
- **88.4%が「FAXやめたい」**けど、63.0%が「取引先がFAXなので移行できない」
- システム入力は60.4%が手入力

## 3. 主要KPI（あるべき姿 vs 現実）

| KPI | あるべき姿 | 現実 |
|-----|----------|------|
| 設備稼働率 | 85%以上 | 計測してない（感覚で「忙しい」） |
| 不良率 | 0.5%以下 | 計測してるが改善PDCAが回ってない |
| 納期遵守率 | 98%以上 | 「だいたい守れてる」（定量把握なし） |
| 見積回答速度 | 24h以内 | 社長次第（3日〜1週間） |
| 新規引合い数 | 月3件以上 | ゼロ（HPから問合せが来たことがない） |
| 原価率 | 正確に把握 | 「多分60%くらい」（正確な原価計算なし） |
| 1人あたり売上 | 2,000万以上 | 計測してない |

## 4. PDCAの停滞パターン

中小製造業で最も多い「PD...止まり」パターン:

```
P (計画): 社長が「来月から原価管理ちゃんとやろう」
D (実行): Excelフォーマット作って1週間は入力する
C (検証): ← ここで止まる（忙しくて振り返る暇がない）
A (改善): ← そもそもCがないのでAも回らない
```

根本原因: 「データが取れてない」のではなく「データはあるけど見てない・活かしてない」

## 7. 理想のPDCAスキーム

```
P: 戦略設計
  ├── ターゲット顧客の明確化（自動車？医療？半導体？）
  ├── 自社の技術的強みの言語化（精度・材質・短納期etc）
  ├── 競合との差別化ポイント整理
  └── HP構成・コンテンツ設計

D: 実行
  ├── HP：加工事例・技術情報を掲載（SEO対策）
  ├── 営業：HP経由の引合い → 見積回答の高速化
  ├── 生産：クラウド工程管理で可視化
  └── 原価：案件別原価を正確に把握

C: 検証（月次）
  ├── HP：PV数・問合せ数・CVR
  ├── 営業：引合い→見積→受注の転換率
  ├── 生産：稼働率・不良率・納期遵守率
  └── 財務：案件別粗利・1人あたり売上

A: 改善
  ├── HPコンテンツの追加・修正（検索流入分析）
  ├── 見積テンプレート改善（回答速度UP）
  ├── 工程ボトルネックの解消
  └── 低粗利案件の見直し
```

## 8. 本音ベースのニーズ構造（建前 vs 本音）

製造業の経営者に「御社の課題は？」と聞くと「HP作らなきゃ」と答えることがある。
しかしこれは**建前**であり、本音の優先順位はまったく異なる。

### 経営者が口にする課題（建前）vs 本当に困っていること（本音）

| 建前（口に出しやすい） | 本音（深刻だが言語化しにくい） |
|---------------------|--------------------------|
| 「HPそろそろ作らなきゃ」 | 「HPで何が変わるか分からないから後回し」 |
| 「DXやらなきゃ」 | 「何から手をつければいいか分からない」 |
| 「新規開拓したい」 | 「既存の発注減が怖い。でも営業する時間がない」 |
| 「ITに詳しい人がいない」 | 「1000万のシステム入れて失敗した過去がある」 |
| 「若い人が来ない」 | 「自分（社長）が辞めたら見積できる人がいない」 |

### 投資判断の本音優先順位

```
【最高緊急】（今月のキャッシュに直結）
  1. 人手不足 → 採用・外国人技能実習
  2. 原価管理 → 「この案件、赤字じゃないか？」が見えない
  3. 見積の属人化 → 社長が倒れたら見積が止まる

【高】（半年〜1年で効いてくる）
  4. FAX/Excel脱却 → 取引先がFAXなので自社だけでは解決不能
  5. 既存顧客の発注減 → 営業努力だが打ち手が限られる

【中〜低】（「やるべき」と分かっているが後回し）
  6. HP → 名刺交換で恥ずかしくない程度でいい
  7. SNS/動画 → そもそも発想にない
```

### この構造が意味すること

**HPは「入口」にはなるが「本丸」ではない。**

- YouTube Ep.1でHP Before/Afterをやるのは**視覚的インパクトと共感獲得**のため
- しかし、視聴者の本音は「HPじゃなくて原価管理どうにかしたい」
- Ep.2以降で本丸（原価可視化・見積デジタル化・データPDCA）に踏み込むことで、
  **「この人は表面的なHP屋じゃなく、経営の本質を分かっている」** という信頼を獲得できる

### omoshikuの戦略的ポジショニング

```
Ep.1 HP（入口・映え）
  │  → 視覚的Before/Afterで「おっ」と思わせる
  │  → 記事・Quick Launchへの導線
  │
Ep.2 原価の見える化（本丸①）
  │  → 「この案件、本当に利益出てますか？」
  │  → Excel原価管理の限界を見せる → 改善提案
  │
Ep.3 見積の脱・属人化（本丸②）
  │  → 「社長しか見積できない問題」を構造化
  │  → テンプレ化 + デジタル化の道筋
  │
Ep.4 データPDCA構築（本丸③）
  │  → 稼働率・不良率・納期遵守率を「測る→見る→改善する」
  │  → ダッシュボード構築の実例
  │
  └→ この時点で視聴者の認識:
     「HP屋」ではなく「製造業の経営改善パートナー」
```

## 9. omoshikuとしての提案機会

### 提案レイヤー（二段構え戦略）

> 投資優先順位の詳細はセクション8「本音ベースのニーズ構造」を参照。
> HPは6番目だが、**YouTube的には最も映える入口**。ここから本丸に誘導する。

| レイヤー | 現状の痛み | omoshikuの提案 | 商材 |
|---------|----------|--------------|------|
| HP（入口） | 技術力が外に伝わらない | HP戦略設計WS + AI制作 | Quick Launch ¥50,000 |
| 営業プロセス | 引合いゼロ、社長が全部やる | HP→見積→CRM導線構築 | Growth Partner（月額） |
| データ活用 | Excel属人化、原価不明 | KPIダッシュボード構築 | DXコンサル |
| 組織変革 | PDが回るけどCAが止まる | 認知科学コーチング×月次PDCA | コーチングパッケージ |

### 営業アクションポイント

| アクション | 具体的にやること |
|-----------|---------------|
| YouTube動画 | 「残念な町工場HP、AIで生まれ変わらせてみた」で認知獲得 |
| 記事SEO | 「製造業 HP 作り方」「町工場 ホームページ」で検索流入 |
| X発信 | `#町工場DX` `#中小製造業` タグで業界コミュニティに参加 |
| 展示会 | 地域の製造業展示会でHP診断ブースを出す |
| 紹介 | Quick Launch実績ができたら、その顧客の取引先に横展開 |
| 補助金連携 | 小規模事業者持続化補助金でHP制作費を補助対象にする |

## 11. 論点（確定事項 + 未決事項）

### 確定事項
- **Ep.1はHP Before/After**（視覚インパクト + 広い共感）→ 製造業「丸山製作所」
- **Ep.2以降で本丸**（原価管理・見積属人化・データPDCA）に踏み込む
- **HPは入口であって目的ではない** — この認識をチーム内で共有すること

### 未決事項
1. YouTube視聴者層を考えると、売上8億（35名）より**もう少し小規模（1-3億・10-15名）の方が共感幅が広い？**
2. Ep.1動画内でDX要素（原価管理・見積属人化）に**チラ見せ**をどこまでやるか
3. 補助金（持続化補助金）への言及を動画内でするか
4. omoshikuのサービスラインナップを製造業向けに拡張するか（DXコンサル商材の具体設計）
5. Ep.2以降の制作優先順位（原価 → 見積 → PDCA の順？）
