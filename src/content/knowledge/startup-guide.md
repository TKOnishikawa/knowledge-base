# 創業準備ガイド - IT/コンサル個人事業主の完全ロードマップ

本ガイドは、IT/コンサルティング分野で **個人事業主（フリーランス）** として独立する際に必要なタスクを全12セクションに体系化したものである。退職準備から法人化判断まで、時系列順に整理している。

**想定読者**: 会社員として IT エンジニアまたはコンサルタントの経験があり、個人事業主として独立を検討している人。

**全体タイムライン（目安）**:

| フェーズ | 時期 | 主なタスク |
|---------|------|-----------|
| 退職準備 | 退職3~6ヶ月前 | 競業避止確認、資金準備、引継ぎ |
| 開業手続き | 退職後1~2週間 | 開業届、青色申告、社会保険切替 |
| 事業立ち上げ | 開業後1ヶ月 | 口座開設、会計、営業開始 |
| 安定化 | 開業後3~6ヶ月 | 契約安定、リピート獲得 |
| 法人化検討 | 所得800万超 | 税メリット試算 |

## 1. 退職準備

> **WHY THIS MATTERS**: 退職の仕方を間違えると、独立前に競業避止違反で訴えられたり、社会保険の空白期間ができたり、退職金・有給を失ったりする。円満退職が独立成功の第一歩。

### タスク一覧

#### 1-1. 退職時期の決定
- **手順**: 退職希望日の最低1ヶ月前（就業規則を確認、多くは1~3ヶ月前の申告が必要）に直属の上司に口頭で伝える。退職届は会社のフォーマットがあればそれに従う。民法上は2週間前の通知で退職可能だが、円満退職のため就業規則に従うのが現実的。
- **ポイント**: 退職日は月末にするのが社会保険上有利（月末退職なら、その月分まで会社が社会保険料を折半負担してくれる）。月末の1日前退職だと、その月の保険料は全額自己負担になる場合がある。

#### 1-2. 競業避止義務の確認
- **手順**: 入社時の雇用契約書、就業規則、秘密保持契約書を確認する。「退職後○年間、同業他社での就業を禁止」のような競業避止条項があるか確認。
- **ポイント**: 日本の判例では、競業避止義務の有効性は「期間の合理性（通常1~2年）」「地域的制限」「代償措置の有無」「業種の限定範囲」で判断される。過度に広い範囲の競業避止は無効になることが多いが、トラブル回避のため弁護士に相談するのが安全。特にコンサル系は顧客情報に関する守秘義務が厳格なので要注意。

#### 1-3. 引継ぎ計画の作成
- **手順**: 退職の意思表示後、担当業務の棚卸しを行い、引継ぎ書を作成する。後任者が決まったら、最低2週間は OJT（実業務を一緒に行う）期間を設ける。
- **ポイント**: 丁寧な引継ぎは将来のリファラル（前職からの紹介案件）につながる。独立後、前職の同僚や上司から仕事を紹介されるケースは非常に多い。

#### 1-4. 有給休暇の消化
- **手順**: 残有給日数を人事に確認し、退職日から逆算して有給消化スケジュールを立てる。有給消化期間中に開業準備（口座開設、名刺作成など）を進めるのが効率的。
- **ポイント**: 有給の買い取りは法的義務ではないため、基本は「使い切る」方針で。有給消化中も在籍扱いなので社会保険は継続される。この期間中に次のセクション以降の準備を進めるのが理想。

#### 1-5. 退職関連書類の受け取り
- **手順**: 退職時に以下の書類を必ず受け取る。
  - **離職票**（ハローワークでの失業手当申請に必要。個人事業を始める場合でも受け取っておく）
  - **源泉徴収票**（確定申告で必要）
  - **健康保険資格喪失証明書**（国民健康保険への切替に必要）
  - **年金手帳**（基礎年金番号の確認用。マイナンバーで代替可能な場合もある）
  - **雇用保険被保険者証**
- **ポイント**: 離職票は退職後2週間程度で届くことが多い。届かない場合は会社の人事に催促する。

### つまづきポイント
- **任意継続の14日期限を忘れる**: 健康保険の任意継続は退職日の翌日から14日以内に手続きが必要。これを過ぎると加入できない（セクション3で詳述）。
- **競業避止条項の確認忘れ**: 退職後に前職から警告書が届いてトラブルになるケース。事前に必ず確認。
- **退職のタイミングミス**: ボーナス支給日前に退職してしまう。支給日在籍条件を就業規則で確認すること。

### コスト・期間
| 項目 | 目安 |
|------|------|
| コスト | 0円（弁護士相談する場合は 5,000~10,000円/30分） |
| 所要期間 | 退職の意思決定から実際の退職まで 1~3ヶ月 |

### 公式リンク

- [厚生労働省 - 退職時の手続き](https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/koyou_roudou/roudoukijun/index.html)

### チェックリスト

- [ ] 競業避止義務の範囲を確認した（必要なら弁護士に相談）
- [ ] 引継ぎ計画を作成し、後任者に共有した
- [ ] 有給休暇の残日数を確認し、消化計画を立てた

- [ ] 退職日が社会保険上有利な日付（月末）であることを確認した

## 2. 開業届・税務関連

> **WHY THIS MATTERS**: 開業届と青色申告承認申請書の提出は、年間最大65万円の所得控除に直結する。期限を過ぎると1年間白色申告になり、数十万円の税金を余分に払うことになる。

### タスク一覧

#### 2-1. 個人事業の開業届出書の提出
- **手順**: 管轄の税務署に「個人事業の開業・廃業等届出書」を提出する。事業開始日から **1ヶ月以内** が提出期限。e-Tax（電子申告）でオンライン提出も可能。
  1. [国税庁のサイト](https://www.nta.go.jp/taxes/tetsuzuki/shinsei/annai/shinkoku/annai/04.htm) から様式をダウンロード（またはe-Taxで作成）
  2. 屋号、事業の概要（「情報処理サービス業」「経営コンサルタント業」など）、開業日を記入
  3. 税務署の窓口に持参または郵送（控えに受領印をもらうこと。口座開設時に必要）
- **ポイント**: 開業届の控え（受領印付き）は、銀行口座開設・融資申請・補助金申請で頻繁に使う。e-Tax で提出した場合は「受信通知」を保存しておく。

#### 2-2. 所得税の青色申告承認申請書の提出
- **手順**: 開業届と同時に提出するのが最も安全。提出期限は **開業日から2ヶ月以内**（1月1日~1月15日に開業した場合はその年の3月15日まで）。
  1. [国税庁のサイト](https://www.nta.go.jp/taxes/tetsuzuki/shinsei/annai/shinkoku/annai/09.htm) から様式をダウンロード
  2. 「複式簿記」を選択（65万円控除の条件）
  3. 備付帳簿は「総勘定元帳」「仕訳帳」にチェック
- **ポイント**: 青色申告の最大のメリットは **65万円の特別控除**（e-Taxで申告した場合。紙提出は55万円）。他にも赤字の3年繰越し、家族への給与の経費算入（青色事業専従者給与）などのメリットがある。

#### 2-3. インボイス制度（適格請求書発行事業者）への登録判断
- **手順**: 取引先が法人（課税事業者）の場合、インボイス登録をしないと取引先が仕入税額控除を受けられなくなるため、取引上不利になる可能性がある。
  1. 取引先にインボイス登録の要否を確認する
  2. 登録する場合は [e-Taxまたは郵送で「適格請求書発行事業者の登録申請書」](https://www.nta.go.jp/taxes/shiraberu/zeimokubetsu/shohi/keigenzeiritsu/invoice_about.htm) を提出
  3. 登録番号（T + 13桁の数字）が通知される（申請から2~3週間）
- **ポイント**:
  - **売上1,000万円以下** の場合、本来は消費税免税事業者だが、インボイス登録すると課税事業者になり消費税の納税義務が生じる
  - **2割特例**（2026年9月まで延長）: インボイス登録に伴い課税事業者になった場合、納税額を売上税額の2割にできる経過措置がある
  - BtoB メインなら登録推奨、BtoC メインなら不要な場合が多い
  - 迷ったら税理士に相談。初年度は登録せず様子を見るのも一つの戦略

#### 2-4. 事業開始等届出書（都道府県税事務所）
- **手順**: 都道府県によっては、都道府県税事務所にも開業届の提出が必要。東京都の場合は「事業開始等届出書」を都税事務所に提出（事業開始日から15日以内）。
- **ポイント**: 国税（税務署）と地方税（都道府県税事務所）は別の組織。提出を忘れても罰則は実質ないが、出しておくのが望ましい。

### つまづきポイント
- **青色申告の期限切れ**: 開業届は出したが青色申告承認申請書を忘れ、初年度は白色申告に。65万円の控除を丸1年逃すことになる。**開業届と同日に提出** が鉄則。
- **インボイスの判断先送り**: 取引先との契約直前にインボイス登録を求められ、登録が間に合わない。事前に主要取引先の方針を確認しておく。
- **開業届の控えを失くす**: 銀行口座開設時に提示を求められて困る。PDF化してクラウド保存を推奨。

### コスト・期間
| 項目 | 目安 |
|------|------|
| コスト | 0円（すべて無料で申請可能） |
| 所要期間 | 書類作成 30分~1時間、税務署への提出 即日 |
| 青色申告の節税効果 | 年間 約10~20万円（所得税率による） |

### 公式リンク
- [国税庁 - 個人事業の開業届出](https://www.nta.go.jp/taxes/tetsuzuki/shinsei/annai/shinkoku/annai/04.htm)
- [国税庁 - 青色申告承認申請書](https://www.nta.go.jp/taxes/tetsuzuki/shinsei/annai/shinkoku/annai/09.htm)
- [国税庁 - インボイス制度](https://www.nta.go.jp/taxes/shiraberu/zeimokubetsu/shohi/keigenzeiritsu/invoice_about.htm)
- [e-Tax（国税電子申告・納税システム）](https://www.e-tax.nta.go.jp/)

## 3. 社会保険・年金

> **WHY THIS MATTERS**: 会社員時代は社会保険料の半分を会社が負担していた。独立すると全額自己負担になり、保険料が大幅に増える可能性がある。最適な選択をしないと年間数十万円の差が出る。

### タスク一覧

#### 3-1. 健康保険の切替（国保 vs 任意継続）
- **手順**: 退職後の健康保険は以下の3つから選択する。
  - **国民健康保険（国保）**: 市区町村の役所で加入手続き。退職日翌日から14日以内。前年の所得に基づいて保険料が決まる。
  - **任意継続**: 退職前の健康保険を最長2年間継続。**退職日の翌日から14日以内** に手続き必要（協会けんぽの場合は各都道府県支部、組合健保の場合はその組合に申請）。保険料は退職時の標準報酬月額に基づく（上限あり）。
  - **家族の扶養に入る**: 年収130万円未満（見込み）の場合のみ。独立して稼ぐ場合は通常該当しない。
- **ポイント**:
  - **比較のコツ**: 市区町村の窓口で国保の保険料を試算してもらい、任意継続の保険料と比較する。前年度の所得が高い場合は任意継続の方が安いケースが多い。逆に前年度の所得が低い場合は国保の方が安い。
  - 任意継続は途中で国保に切り替えることが可能（2022年の法改正で任意の脱退が可能になった）。逆（国保→任意継続）はできない。
  - 国保には扶養の概念がないため、家族がいる場合は家族分も保険料がかかる。任意継続なら家族を扶養に入れられる。

#### 3-2. 国民年金への切替
- **手順**: 市区町村の役所で厚生年金から国民年金（第1号被保険者）への切替手続きを行う。退職日の翌日から14日以内。年金手帳（または基礎年金番号通知書）とマイナンバーカードを持参。
- **ポイント**: 国民年金保険料は月額 **16,980円**（2025年度。毎年度改定）。前納制度を使うと割引がある（2年前納で約15,000円割引）。支払いが厳しい場合は免除・猶予制度もあるが、将来の年金額が減るので注意。

#### 3-3. 国民年金基金・iDeCo・小規模企業共済の検討
- **手順**: 会社員時代の厚生年金に比べ、国民年金だけでは将来の年金額が大幅に減る。以下の上乗せ制度を検討する。
  - **iDeCo（個人型確定拠出年金）**: 掛金が全額所得控除。月額最大68,000円（国民年金基金と合算）。60歳まで引き出せない。
  - **国民年金基金**: 掛金が全額所得控除。iDeCoと合算で月額68,000円まで。終身年金を上乗せできる。
  - **小規模企業共済**: 個人事業主の「退職金」制度。掛金が全額所得控除。月額1,000~70,000円。事業廃止時に一括受取可能。
- **ポイント**: 節税効果が高い順に **小規模企業共済 > iDeCo > 国民年金基金**（流動性を考慮した場合）。所得が安定してから加入を検討するのが現実的。小規模企業共済は掛金を柔軟に変更できるため、まずは月1,000円から始めるのもあり。

#### 3-4. 労災保険の特別加入（任意）
- **手順**: IT系フリーランスも「特別加入」で労災保険に加入可能（2021年9月から対象範囲拡大）。労働保険事務組合を通じて申請する。
- **ポイント**: 月額数千円で業務中の事故・病気に対する補償が受けられる。特に客先常駐型の働き方をする場合は検討に値する。

### つまづきポイント
- **14日期限を忘れる**: 任意継続の手続きは退職日の翌日から14日以内。1日でも過ぎると加入不可。退職日が決まったらすぐに申請書を準備すること。
- **保険料の急増に驚く**: 会社員時代は会社が半分負担していたため、実質負担額は倍になる。年収600万円の場合、国保で年間40~60万円程度（自治体による）。事前にシミュレーションしておく。
- **手続きの空白期間**: 退職から手続き完了までの間、保険証がない期間ができる。その間に病院にかかると一旦10割負担になる（後日還付は可能）。

### コスト・期間
| 項目 | 目安 |
|------|------|
| 国民健康保険料 | 年収600万円で年間 40~60万円（自治体により大きく異なる） |
| 任意継続保険料 | 月額 2~4万円程度（上限あり、標準報酬月額30万円の上限が多い） |
| 国民年金保険料 | 月額 16,980円（2025年度） |
| 小規模企業共済 | 月額 1,000~70,000円（全額所得控除） |
| iDeCo | 月額 最大68,000円（全額所得控除） |
| 所要期間 | 役所での手続き 1~2時間（待ち時間含む） |

## 4. 事業計画

> **WHY THIS MATTERS**: 「なんとなくフリーランス」では単価が上がらず、案件が途切れた時に方向性を見失う。サービス定義と料金設計を明確にすることが安定収入の土台。

### タスク一覧

#### 4-1. サービス定義（何を提供するか）
- **手順**: 自分のスキル・経験を棚卸しし、提供するサービスを明確に定義する。
  1. **スキル棚卸し**: 技術スキル（言語、フレームワーク、クラウド）、業務スキル（PM、要件定義、コンサル）、業界知識を洗い出す
  2. **サービスメニュー化**: 「システム開発」だけでなく、「クラウドインフラ設計・構築（AWS/GCP）」「DX推進コンサルティング」「データ分析基盤構築」のように具体化する
  3. **提供形態の決定**: 準委任（時間精算）か請負（成果物納品）か、常駐か リモートかを決める
- **ポイント**: IT系は「準委任契約（月額精算）」、コンサル系は「アドバイザリー契約（月額固定 or 日額）」が主流。準委任は収入が安定しやすく、請負は単価が高くなる傾向がある。

#### 4-2. 料金設計
- **手順**: 市場相場を調査し、自分のスキルレベルに合った料金を設定する。
  - **IT エンジニア（準委任・月額）**:
    - ジュニア（経験3年未満）: 月50~70万円
    - ミドル（経験3~7年）: 月70~100万円
    - シニア（経験7年以上）: 月80~150万円
    - PM/アーキテクト: 月100~180万円
  - **コンサルタント（日額 or 月額）**:
    - 日額: 5~15万円（経験・専門性による）
    - 月額アドバイザリー: 30~80万円（稼働日数による）
  - **スポット案件（請負）**: プロジェクト規模に応じて個別見積り
- **ポイント**: 初期は相場の中間で始め、実績を積んだら段階的に単価を上げる。会社員時代の年収 ÷ 12 を「最低ライン」とし、それに社会保険料・経費・利益を上乗せして単価を計算する。

#### 4-3. ターゲット顧客の定義
- **手順**: どのような企業・業界をターゲットにするか明確にする。
  1. 前職の経験から強みを発揮できる業界を特定
  2. 企業規模（スタートアップ / 中小 / 大企業）と意思決定フローを理解
  3. 理想の顧客像（ペルソナ）を1~2パターン作成
- **ポイント**: 最初はニッチに絞る方が差別化しやすい。「なんでもできます」は「何も強みがない」と同義。

#### 4-4. 売上目標の設定
- **手順**: 年間の売上目標を設定し、逆算で必要な案件数・稼働率を計算する。
  - 例: 年間売上目標 1,000万円 → 月額83万円 → 月単価100万円で稼働率85%
  - 稼働率（ビラブルレシオ）は **75~85%** を目安に（営業・管理・休暇で15~25%は非稼働）
- **ポイント**: 初年度は稼働率60~70%で計算するのが現実的（営業活動に時間がかかるため）。

### つまづきポイント
- **安売りしてしまう**: 独立直後の不安から相場より大幅に安い単価で受注し、疲弊する。最低ラインを事前に決めておく。
- **差別化不足**: 「Javaができます」だけでは埋もれる。「金融業界のシステムリプレース経験10年」のように、技術 × 業界知識で差別化する。
- **事業計画を作らない**: 頭の中だけで考え、状況が変わった時に判断基準がなくなる。A4一枚でも良いので文書化する。

### コスト・期間
| 項目 | 目安 |
|------|------|
| コスト | 0円（自分で作成する場合） |
| コンサル依頼 | 5~30万円（事業計画作成支援） |
| 所要期間 | 1~2週間（じっくり考える時間含む） |

### 公式リンク
- [中小企業庁 - 事業計画策定ガイドライン](https://www.chusho.meti.go.jp/)
- [J-Net21 - 起業マニュアル](https://j-net21.smrj.go.jp/startup/manual/index.html)

- [ ] 提供サービスを具体的に定義した（サービスメニュー）
- [ ] 料金体系を設定した（最低ラインを決定）

## 5. 資金計画・融資

> **WHY THIS MATTERS**: 独立後の収入は不安定になる。最低6ヶ月分の生活費と事業資金を確保しておかないと、案件が途切れた時に資金ショートする。また、融資は「在職中」の方が圧倒的に有利。

### タスク一覧

#### 5-1. 初期資金の算出
- **手順**: 独立に必要な初期資金を算出する。IT/コンサル系は設備投資が少ないため、比較的少額で始められる。
  - **最低限の初期資金（IT/コンサル）**:
    - PC・周辺機器: 15~30万円（既に持っている場合は0円）
    - ソフトウェア・SaaS: 年間5~15万円
    - 名刺・ロゴ・Webサイト: 3~10万円
    - 予備費: 10~20万円
    - **合計: 50~100万円**
  - **生活防衛資金**: 月の生活費 × 6ヶ月分（家賃・食費・保険料・年金など）
- **ポイント**: 生活防衛資金と事業資金は別で管理すること。初期は節約志向で始め、収入が安定してから投資を増やす。

#### 5-2. 日本政策金融公庫の融資
- **手順**: 政府系金融機関である日本政策金融公庫の「新規開業資金」が最も利用しやすい。
  1. 最寄りの支店に相談予約（電話 or Web）
  2. 創業計画書を作成（公庫の書式あり）
  3. 必要書類を準備（開業届、事業計画書、通帳のコピー、見積書など）
  4. 面談（1~2時間）
  5. 審査（2~3週間）
  6. 融資実行
  - **融資限度額**: 最大7,200万円（うち運転資金4,800万円）
  - **金利**: 年1.0~2.5%程度（条件による）
  - **無担保・無保証人** の制度あり
- **ポイント**:
  - **在職中に申し込む** のが有利（安定収入がある状態で信用力が高い）
  - 自己資金は借入希望額の **1/3以上** が目安（審査で重視される）
  - 創業計画書は具体的に（「IT企業向けコンサル、月3社、月額50万円」のように数字で示す）

#### 5-3. IT導入補助金
- **手順**: 中小企業・小規模事業者がITツールを導入する際に費用の一部が補助される。
  - **通常枠**: 補助率 1/2、補助額 1万~150万円
  - **インボイス枠**: 補助率 3/4~4/5、補助額 最大350万円
  - 対象: IT導入支援事業者が登録したITツール（会計ソフト、受発注システムなど）
- **ポイント**: 自分で開発に使うツール（IDE、クラウドサービスなど）は対象外のものが多い。会計ソフト（freee、マネーフォワード）やセキュリティツールが対象になりやすい。

#### 5-4. 小規模事業者持続化補助金
- **手順**: 販路開拓等の取り組みに対して最大50~200万円が補助される。
  - **一般型**: 補助率 2/3、補助上限 50万円
  - **創業枠**: 補助上限 200万円（産業競争力強化法に基づく認定市区町村の創業支援等事業による支援を受けた事業者）
  - 対象: Webサイト制作、チラシ・名刺作成、展示会出展、広告費など
- **ポイント**: 採択率は40~60%程度。申請書の「経営計画書」「補助事業計画書」の出来が重要。商工会・商工会議所の支援を受けながら作成するのが一般的。

#### 5-5. その他の資金調達
- **手順**: 状況に応じて以下も検討する。
  - **信用金庫・信用組合**: 地域密着型で小規模事業者に比較的融資しやすい
  - **自治体の制度融資**: 自治体が利子補給や信用保証料を補助。金利が低い
  - **クレジットライン**: 事業用クレジットカードの活用（セクション7で詳述）
- **ポイント**: 融資と補助金は併用可能。補助金は「後払い」（先に自己資金で支出し、後日補助金が入金される）なので、資金繰りに注意。

### つまづきポイント
- **退職後に融資を申し込む**: 会社員時代に比べて信用力が大幅に下がり、融資のハードルが上がる。在職中から準備を始めること。
- **自己資金が少なすぎる**: 借入希望額の1/3未満だと審査で不利になる。最低100万円程度の自己資金は貯めておく。
- **補助金を当てにした資金計画**: 補助金は不採択になる可能性がある。補助金なしでも事業が回る計画を基本とする。

### コスト・期間
| 項目 | 目安 |
|------|------|
| 初期資金（IT/コンサル） | 50~100万円 |
| 生活防衛資金 | 月の生活費 × 6ヶ月分 |
| 日本政策金融公庫 融資 | 審査 2~3週間、金利 年1.0~2.5% |
| IT導入補助金 | 補助率 1/2~4/5（申請から交付まで3~6ヶ月） |
| 小規模事業者持続化補助金 | 補助上限 50~200万円（年に数回公募） |

### 公式リンク
- [日本政策金融公庫 - 新規開業資金](https://www.jfc.go.jp/n/finance/search/01_sinkikaigyou_m.html)
- [日本政策金融公庫 - 創業計画書テンプレート](https://www.jfc.go.jp/n/service/dl_kokumin.html)
- [IT導入補助金](https://it-shien.smrj.go.jp/)
- [小規模事業者持続化補助金](https://r3.jizokukahojokin.info/)
- [J-Net21 - 補助金・助成金情報](https://j-net21.smrj.go.jp/subsidy/index.html)

### チェックリスト
- [ ] 初期資金と生活防衛資金の必要額を算出した
- [ ] 自己資金が十分か確認した（借入希望額の1/3以上）
- [ ] 日本政策金融公庫への相談を行った（または不要と判断した）
- [ ] IT導入補助金の対象になるか確認した

## 6. 屋号・ブランディング

> **WHY THIS MATTERS**: 屋号やブランドは「あなたが何者か」を伝える最初の接点。ドメインやSNSアカウントの先取りを放置すると、後から確保できなくなる。

### タスク一覧

#### 6-1. 屋号の決定
- **手順**:
  1. 屋号の候補を5~10個リストアップ
  2. 以下を確認:
     - 商標登録されていないか（[J-PlatPat](https://www.j-platpat.inpit.go.jp/) で検索）
     - ドメインが取得可能か（お名前.com、ムームードメインなどで検索）
     - SNSアカウント（X、LinkedIn、Instagram）が取得可能か
     - 同名の法人が存在しないか（[国税庁法人番号公表サイト](https://www.houjin-bangou.nta.go.jp/) で検索）
  3. 開業届の屋号欄に記入（後からの変更も可能だが、手間がかかる）
- **ポイント**: IT/コンサル系の屋号は「英語名（カタカナ読み付き）」が多い。覚えやすく、業種がイメージできる名前が理想。「○○コンサルティング」「○○テクノロジーズ」「○○ラボ」など。屋号は法人名と違い登記が不要で、自由に付けられる。

#### 6-2. ドメイン取得
- **手順**: 屋号に合わせた独自ドメインを取得する。
  1. `.com` `.jp` `.co.jp`（法人のみ）の優先順で確認
  2. 年間費用: `.com` で約1,500円/年、`.jp` で約3,000円/年
  3. Whois代行を有効にしてプライバシーを保護
- **ポイント**: メールアドレスにも独自ドメインを使うとプロフェッショナルな印象になる（例: `info@yourcompany.com`）。Google Workspace（月額680円~）やさくらのメールボックス（月額87円~）でドメインメールを運用可能。

#### 6-3. 名刺の作成
- **手順**:
  1. 記載情報: 屋号、氏名、肩書、電話番号、メールアドレス、Webサイト、住所（自宅の場合は番地まで書かないことも検討）
  2. デザイン: Canva（無料プランあり）で自作するか、ラクスルなどの印刷サービスを利用
  3. 印刷: 100枚 1,000~3,000円程度
- **ポイント**: 住所は自宅をそのまま載せるのに抵抗がある場合、バーチャルオフィスの住所を使う方法もある（月額500~5,000円）。QRコード付きで、ポートフォリオサイトに誘導するのも効果的。

#### 6-4. ロゴの作成
- **手順**:
  - **自作**: Canva、Figma で作成（無料）
  - **外注**: ココナラ（3,000~30,000円）、クラウドワークス、99designs
  - **AI生成**: Midjourney、DALL-E（商用利用の規約確認が必要）
- **ポイント**: 初期は凝ったロゴは不要。屋号のテキストロゴで十分。事業が軌道に乗ってからリブランディングすればよい。

#### 6-5. SNSアカウント・Webサイトの確保
- **手順**:
  1. 以下のプラットフォームで屋号名のアカウントを確保:
     - **LinkedIn**（BtoB の信頼構築に必須）
     - **X（Twitter）**（IT業界での情報発信）
     - **GitHub**（エンジニアの場合、ポートフォリオ代わり）
  2. Webサイト: 最低限のランディングページを作成
     - 簡易: Notion + 独自ドメイン（無料~月額8ドル）
     - 本格: WordPress / Astro / Next.js + Vercel（無料~月額数千円）
- **ポイント**: 発信を始めるのは早ければ早いほど良い。在職中からブログやSNSで専門知識を発信しておくと、独立後の集客につながる。

### つまづきポイント
- **商標衝突**: 屋号を使い始めてから商標権侵害の警告を受ける。事前にJ-PlatPatで検索すること。
- **ドメインの先取り**: 思いついた屋号のドメインが既に取られている。屋号決定と同時にドメインを取得する。
- **個人情報の露出**: 名刺やWebサイトに自宅住所・電話番号を載せてしまい、プライバシーの問題に。バーチャルオフィスやIP電話の活用を検討。

### コスト・期間
| 項目 | 目安 |
|------|------|
| ドメイン取得 | 年間 1,500~3,000円 |
| 名刺作成 | 1,000~3,000円（100枚） |
| ロゴ外注 | 3,000~30,000円（ココナラ） |
| Webサイト | 0~5,000円/月 |
| バーチャルオフィス | 月額 500~5,000円 |
| 所要期間 | 1~2週間 |

- [国税庁 法人番号公表サイト](https://www.houjin-bangou.nta.go.jp/)
- [Canva](https://www.canva.com/)

- [ ] 商標の衝突がないことを確認した（J-PlatPat）

- [ ] ロゴを作成した（シンプルでOK）

## 7. 銀行口座・クレジットカード

> **WHY THIS MATTERS**: 個人の口座と事業の口座を分けないと、確定申告の時に地獄を見る。また、開業直後は口座開設の審査が厳しいため、タイミングが重要。

### タスク一覧

#### 7-1. 事業用銀行口座の開設
- **手順**: 個人事業主は「個人名義 + 屋号」の口座を開設する。
  - **ネット銀行（推奨）**:
    - **住信SBIネット銀行**: 屋号付き口座対応。振込手数料が安い（月数回無料）。API連携が充実
    - **楽天銀行**: 個人ビジネス口座。楽天経済圏との連携。振込手数料 145円~
    - **PayPay銀行**: ビジネスアカウント対応。ヤフー系サービスとの連携
    - **GMOあおぞらネット銀行**: freee との API 連携が強い。振込手数料 145円
  - **メガバンク・地銀**:
    - 信頼性が高いが、振込手数料が高く、屋号付き口座の開設ハードルが高い場合がある
    - 融資を受ける場合はメインバンクとの取引実績が有利に働く
  - **ゆうちょ銀行**: 振替口座は屋号のみで開設可能。手数料が安い
- **ポイント**:
  - **開業届の控え（受領印付き）** が口座開設時に求められることが多い
  - ネット銀行は開設が比較的容易で、会計ソフトとのAPI連携が便利
  - 最低でも2つ（メイン + 税金用積立）の口座を持つと管理しやすい

#### 7-2. 事業用クレジットカードの作成
- **手順**: 事業経費の支払い用にビジネスカードを作成する。
  - **おすすめのビジネスカード（個人事業主向け）**:
    - **freee カード**: 年会費無料。freee と完全連携で仕訳が自動化
    - **三井住友カード ビジネスオーナーズ**: 年会費無料。最大500万円の限度額
    - **アメリカン・エキスプレス・ビジネス・カード**: 年会費13,200円。ステータスと付帯サービス充実
    - **楽天ビジネスカード**: 年会費2,200円。楽天プレミアムカード（年会費11,000円）との併用必須
  - 審査は **個人の信用情報** に基づくため、退職前（会社員の信用力があるうち）に申し込むのが有利
- **ポイント**:
  - 事業の経費はすべてこのカードで支払い、個人のカードとは完全に分離する
  - 会計ソフトとのカード連携で、利用明細が自動取り込みされて仕訳が楽になる

#### 7-3. 決済手段の整備
- **手順**: クライアントからの入金方法と、自分の支払い方法を整備する。
  - **請求書払い**: 銀行振込が基本（IT/コンサルのBtoB取引は99%がこれ）
  - **オンライン決済**: Stripe、Square、PayPal（小規模案件やBtoC向け）
  - **電子帳簿保存法対応**: 電子取引データ（メール請求書、カード明細など）の保存ルールを確認
- **ポイント**: 振込手数料の負担は契約時に明確にしておく（「振込手数料は貴社ご負担でお願いします」が一般的）。

### つまづきポイント
- **開業届なしで口座開設を申し込む**: ネット銀行では開業届の控えを求められることが多い。開業届を先に出しておく。
- **個人口座で事業取引を始めてしまう**: 後から分離するのは非常に手間。初日から事業用口座を使う。
- **退職後にクレジットカードを申し込む**: 審査で不利。在職中に1枚は作っておく。

## 8. 会計・経理体制

> **WHY THIS MATTERS**: 青色申告65万円控除を受けるには複式簿記が必須。手作業で帳簿を付けるのは非現実的なので、会計ソフトの導入は事実上必須。初月から正しく記帳することで確定申告時の地獄を回避できる。

### タスク一覧

#### 8-1. 会計ソフトの選定・導入
- **手順**: 以下の3大クラウド会計ソフトから選択する。

| 比較項目 | freee | マネーフォワード クラウド確定申告 | やよいの青色申告 オンライン |
|---------|-------|--------------------------|------------------------|
| 月額料金 | 1,180円~（スターター） | 980円~（パーソナルミニ） | 初年度無料~（セルフプラン） |
| 銀行連携 | 強い（API連携多数） | 強い（API連携多数） | 普通 |
| 操作性 | 簿記知識なくても使える | やや簿記知識が必要 | 伝統的な帳簿UI |
| 請求書機能 | 内蔵（追加料金なし） | 別サービス（クラウド請求書） | 別サービス |
| インボイス対応 | 対応済み | 対応済み | 対応済み |
| 確定申告 | e-Tax連携あり | e-Tax連携あり | e-Tax連携あり |
| おすすめ | 簿記知識がない人 | 簿記の基礎がある人 | コストを抑えたい人 |

- **ポイント**: IT/コンサル系の個人事業主には **freee** が最もおすすめ。簿記知識がなくても仕訳ができ、請求書発行・入金管理・確定申告まで一気通貫で行える。初年度は各社とも無料トライアルがあるので、試してから決めるのも可。

#### 8-2. 勘定科目の理解
- **手順**: IT/コンサル系でよく使う勘定科目を理解しておく。

| 勘定科目 | 内容 | 具体例 |
|---------|------|--------|
| 売上高 | 事業の収入 | コンサル報酬、開発費 |
| 通信費 | ネット、電話 | インターネット回線、携帯電話料金 |
| 消耗品費 | 10万円未満の備品 | キーボード、モニター、USBメモリ |
| 減価償却費 | 10万円以上の備品 | PC（4年で償却）、ソフトウェア（5年） |
| 旅費交通費 | 移動費 | 電車賃、タクシー代、出張の宿泊費 |
| 接待交際費 | 取引先との飲食 | クライアントとの食事、手土産 |
| 新聞図書費 | 書籍・サブスク | 技術書、Udemy、有料ニュース |
| 研修費 | スキルアップ | セミナー参加費、資格試験 |
| 地代家賃 | 作業場所 | コワーキングスペース月額 |
| 支払手数料 | 各種手数料 | 振込手数料、クラウドソーシング手数料 |
| 租税公課 | 税金関連 | 個人事業税、印紙代 |
| 外注費 | 業務の一部外注 | デザイン外注、コーディング外注 |

- **ポイント**: 迷ったら「消耗品費」に入れておき、後で税理士に確認するのが安全。重要なのは「同じ性質の支出は同じ科目に統一する」こと。

#### 8-3. 請求書テンプレートの作成
- **手順**: インボイス制度に対応した請求書を作成する。記載必須事項:
  1. 発行者の氏名（屋号）
  2. **適格請求書発行事業者の登録番号**（T + 13桁。インボイス登録した場合）
  3. 取引日
  4. 取引内容
  5. 税率ごとの対価の額と適用税率
  6. 税率ごとの消費税額
  7. 受領者の氏名
- **ポイント**: freeeやマネーフォワードの請求書機能を使えば、上記要件を自動で満たした請求書が発行できる。独自で作る場合はExcelテンプレートを整備する。

#### 8-4. 記帳のルーティン化
- **手順**:
  - **毎日**: 経費の領収書を写真撮影してクラウドに保存（freeeアプリ等）
  - **毎週**: 銀行口座・カード明細と会計ソフトの自動取り込みを確認・仕訳
  - **毎月**: 売上・経費の月次集計、請求書の発行・入金確認
  - **毎年**: 確定申告（翌年2月16日~3月15日。e-Taxなら1月から提出可能）
- **ポイント**: 溜めると確定申告直前に地獄を見る。週1回の仕訳確認を習慣にする。

#### 8-5. 税理士への依頼判断
- **手順**: 以下の場合は税理士への依頼を検討する。
  - 年間売上が500万円を超えた
  - 記帳に月2時間以上かかっている
  - 節税対策のアドバイスが欲しい
  - 法人化を検討している
  - 費用目安: 確定申告のみ 5~15万円/年、顧問契約 月1~3万円
- **ポイント**: IT系に強い税理士を選ぶこと（SaaSの経費処理、仮想通貨の取り扱いなどに詳しい）。紹介プラットフォーム: [税理士ドットコム](https://www.zeiri4.com/)、[ミツモア](https://meetsmore.com/)。

### つまづきポイント
- **個人と事業の経費を混同**: 個人の買い物を事業用カードで支払ったり、その逆をしたりする。完全分離が鉄則。「事業主貸」「事業主借」の仕訳で対応は可能だが、頻繁だと混乱する。
- **領収書を捨ててしまう**: 法定保存期間は青色申告で7年。電子帳簿保存法に対応した方法（スキャン保存やクラウド保存）で管理する。
- **確定申告直前に一年分をまとめて記帳**: 記帳漏れ、分類ミスが多発する。月次で締めるのが理想。

## 9. 契約書・法務

> **WHY THIS MATTERS**: 口約束で仕事を始めると、納品後に「聞いていない」「契約条件が違う」と言われてトラブルになる。契約書は「もしもの時の保険」であり、良好な関係の時にこそ作るもの。

### タスク一覧

#### 9-1. 業務委託契約書の整備
- **手順**: クライアントと業務を開始する前に必ず契約書を交わす。以下の項目を含めること。
  1. **業務内容**: 具体的な作業範囲（スコープ）を明記。「その他付随する業務」は範囲を限定する
  2. **契約形態**: 準委任（善管注意義務・月額精算）か請負（完成義務・成果物納品）か
  3. **報酬・支払条件**: 金額、消費税の取り扱い、支払日（例: 月末締め翌月末払い）、振込手数料の負担
  4. **契約期間**: 開始日・終了日、自動更新の有無、中途解約の条件
  5. **知的財産権**: 成果物の著作権の帰属（通常は報酬支払い後にクライアントに移転）
  6. **秘密保持**: 秘密情報の定義、保持期間（通常2~5年）
  7. **損害賠償**: 上限（通常は報酬の○ヶ月分。無制限にしない）
  8. **再委託**: 外注の可否
  9. **反社会的勢力の排除**: 標準条項
  10. **管轄裁判所**: 紛争時の裁判所を指定
- **ポイント**: クライアントから提示された契約書をそのまま受け入れず、必ず内容を確認する。不利な条項（無制限の損害賠償、過度な競業避止、著作権の包括的譲渡など）は交渉すること。

#### 9-2. NDA（秘密保持契約）の整備
- **手順**: 業務委託契約の前段階で、提案段階から秘密情報のやり取りがある場合に締結する。
  - 基本的には業務委託契約書の秘密保持条項で足りるが、別途NDAを求められる場合もある
  - NDAのテンプレートは経済産業省が公開しているものが参考になる
- **ポイント**: NDA は双方向（相互秘密保持）にするのが一般的。一方的にこちらだけ義務を負うNDAは避ける。

#### 9-3. 下請法の理解
- **手順**: 自分が発注側になる場合（外注先に業務を委託する場合）は下請法が適用される可能性がある。
  - **対象**: 資本金1,000万円超の法人が、資本金1,000万円以下の事業者に業務を委託する場合
  - 個人事業主が下注先になる場合は、下請法で保護される
  - **禁止事項**: 不当な値引き、支払遅延（60日以内に支払い）、不当な返品
- **ポイント**: 個人事業主として発注する側になる場合は少ないが、受注する側として「支払いが60日以内か」「不当な値引きがないか」は確認すべき。

#### 9-4. 損害賠償条項の交渉
- **手順**:
  - **上限の設定**: 「直接かつ現実に被った損害に限り、報酬総額の○ヶ月分を上限とする」が一般的
  - **間接損害の除外**: 「逸失利益、間接損害は賠償の対象外とする」
  - **過失責任**: 故意または重過失の場合のみ責任を負う
- **ポイント**: 「損害賠償は無制限」という条項を安易に受け入れない。小さなバグで数千万円の賠償請求をされるリスクがある。個人事業主にとっては事業存続に関わる問題。

#### 9-5. 契約不適合責任（旧: 瑕疵担保）の確認
- **手順**: 請負契約の場合、納品物に不具合（契約不適合）があった場合の責任期間と対応範囲を確認する。
  - 民法上のデフォルト: 発見から1年以内に通知
  - 契約で短縮・延長可能（IT業界では3ヶ月~1年が一般的）
  - 対応範囲: 修補、代金減額、損害賠償
- **ポイント**: 準委任契約の場合は契約不適合責任は原則適用されない（成果物の完成義務がないため）。契約形態の選択が重要。

### つまづきポイント
- **契約書なしで着手**: 「信頼関係があるから」と口約束で始め、後でトラブルになる。どんなに親しい関係でも契約書は交わす。
- **契約不適合責任の期間が長すぎる**: 2年、3年の契約不適合期間を受け入れてしまう。IT業界では3ヶ月~1年が相場。
- **著作権の包括譲渡**: 全著作権（翻案権・二次著作物利用権含む）を無条件で譲渡する契約。ポートフォリオに使えなくなるなど不利益がある。

|------|------|
| 契約書テンプレート作成 | 0円（自作）~10万円（弁護士依頼） |
| 弁護士によるリーガルチェック | 3~10万円/件 |
| 弁護士顧問契約 | 月額 3~5万円 |
| 所要期間 | テンプレート作成 1~3日、リーガルチェック 1~2週間 |

## 10. 営業・集客

> **WHY THIS MATTERS**: 技術力があっても仕事が来なければ売上はゼロ。特に独立直後は「どうやって案件を獲得するか」が最大の課題。複数の集客チャネルを持つことがリスク分散になる。

### タスク一覧

#### 10-1. エージェント（案件紹介サービス）への登録
- **手順**: IT/コンサル系フリーランス向けのエージェントに複数登録する。

| エージェント | 特徴 | マージン | 単価帯 |
|-------------|------|---------|--------|
| **レバテックフリーランス** | 最大手。案件数豊富。リモート案件も多い | 非公開（10~15%程度） | 月60~150万円 |
| **PE-BANK** | 老舗。マージン率が明示されている | 8~12%（実績で下がる） | 月60~120万円 |
| **Midworks** | 給与保証制度あり。福利厚生充実 | 非公開（15~20%程度） | 月50~100万円 |
| **ITプロパートナーズ** | 週2~3日案件が多い。副業にも | 非公開 | 月30~80万円 |
| **ギークスジョブ** | 高単価案件に強い | 非公開 | 月70~150万円 |
| **フォスターフリーランス** | PM/コンサル案件に強い | 非公開 | 月80~160万円 |

  1. 複数のエージェントに登録（3~5社が目安）
  2. 経歴書（スキルシート）を整備
  3. 面談で希望条件を伝える（単価、稼働日数、リモート可否、技術領域）
  4. 案件紹介を受け、企業面談に進む
- **ポイント**: エージェント経由の案件は仲介手数料が引かれるため、手取りが減る。しかし、独立直後は案件を安定的に獲得する手段として非常に有効。エージェントは営業活動を代行してくれるので、本業に集中できる。

#### 10-2. クラウドソーシングの活用
- **手順**:
  - **Lancers / CrowdWorks**: 日本最大級のクラウドソーシング。小~中規模案件が中心
  - **ココナラ**: スキル販売プラットフォーム。「技術相談1時間 5,000円」のようなスポット案件
  - **SOKUDAN / Workship**: 業務委託特化型。エンジニア・コンサル向け高単価案件
- **ポイント**: クラウドソーシングは単価が低い傾向があるが、実績を積む場として有効。プロフィールの充実度と評価が案件獲得に直結する。

#### 10-3. 直営業（ダイレクト営業）
- **手順**: エージェントを介さず、直接企業と契約する方法。マージンがないため手取りが最大化する。
  1. **知人・前職の人脈**: 最も確実な案件獲得方法。退職時の人間関係が重要
  2. **LinkedInでのアウトバウンド**: ターゲット企業の意思決定者にコンタクト
  3. **ブログ・SNS経由の問い合わせ**: 専門知識の発信がインバウンド営業につながる
  4. **交流会・カンファレンス**: IT系の勉強会やビジネス交流会に参加
  5. **紹介（リファラル）**: 既存クライアントからの紹介。最も成約率が高い
- **ポイント**: 直営業は時間がかかるが、エージェント手数料がないため利益率が高い。長期的にはリファラルと直営業の比率を上げていくのが理想。

#### 10-4. ポートフォリオ・実績の整備
- **手順**:
  - GitHubで公開可能なプロジェクトを整備
  - 技術ブログで専門知識を発信（Zenn、Qiita、note、自社サイト）
  - 実績紹介ページを作成（守秘義務に抵触しない範囲で）
  - 得意技術の資格を取得（AWS認定、GCP認定、PMP等）
- **ポイント**: 「何ができるか」を言葉で説明するよりも、「実際にやったこと」を見せる方が100倍効果的。

### つまづきポイント
- **1社依存**: 1つのクライアントに売上の80%以上を依存すると、その案件が終了した時にいきなり収入がゼロになる。常に2~3社と取引がある状態を目指す。
- **単価交渉をしない**: 「言い値で受ける」を続けると、いつまでも単価が上がらない。実績を積んだら定期的に単価交渉する。契約更新のタイミングが交渉しやすい。
- **営業活動を後回しにする**: 目の前の案件に集中しすぎて、次の案件の種まきを忘れる。稼働中も週に数時間は営業・情報発信に充てる。

### コスト・期間
| 項目 | 目安 |
|------|------|
| エージェント登録 | 無料（マージンは案件単価に含まれる） |

## 11. オフィス・インフラ

> **WHY THIS MATTERS**: 自宅で仕事をする場合、家賃やネット代の一部を経費にできる（家事按分）。正しく計算しないと、経費を取りこぼすか、逆に過大計上で税務調査のリスクになる。

### タスク一覧

#### 11-1. 作業場所の選択
- **手順**: 以下の選択肢から、自分のワークスタイルに合った作業場所を選ぶ。

| 選択肢 | メリット | デメリット | 月額コスト |
|--------|---------|-----------|-----------|
| **自宅** | コスト最小、通勤なし | 公私の区別が曖昧 | 0円（家事按分で一部経費化） |
| **コワーキングスペース** | 集中できる環境、人脈形成 | 固定費がかかる | 5,000~30,000円 |
| **バーチャルオフィス** | 住所利用のみ。登記対応可 | 実際の作業場所ではない | 500~5,000円 |
| **賃貸オフィス** | 本格的なオフィス環境 | 固定費が高い | 50,000~200,000円 |

- **ポイント**: IT/コンサル系の個人事業主は **自宅 + 必要に応じてコワーキング** が最もコスパが良い。客先常駐の場合は自宅で十分。名刺やWebに自宅住所を載せたくない場合はバーチャルオフィスを併用する。

#### 11-2. 家事按分の計算
- **手順**: 自宅を作業場所にする場合、以下の経費を按分して事業経費にできる。
  - **家賃**: 面積比（事業用スペースの面積 ÷ 総面積）で按分するのが一般的
    - 例: 60平米の部屋で、10平米を仕事部屋として使用 → 按分率 16.7%
    - 家賃12万円 × 16.7% = 月20,040円が経費
  - **電気代**: 時間比（事業使用時間 ÷ 全使用時間）で按分
    - 例: 1日8時間仕事、起きている時間16時間 → 按分率 50%（仕事時間/在宅時間で計算する方法もある）
  - **インターネット回線**: 事業使用割合で按分（50~70%が一般的）
  - **携帯電話**: 事業使用割合で按分（30~50%が一般的）
  - **水道代**: 按分は認められにくい（IT系では使用量に差が出ないため）
- **ポイント**: 按分比率の根拠を明確にしておくこと（計算根拠をメモに残す）。税務調査で「なぜこの比率なのか」と聞かれた時に説明できる必要がある。

#### 11-3. SaaS・ツールの整備
- **手順**: 事業に必要なSaaSツールを整備する。

| カテゴリ | ツール | 月額 |
|---------|--------|------|
| **コミュニケーション** | Slack（無料プラン可）、Google Meet、Zoom | 0~2,000円 |
| **プロジェクト管理** | Notion（無料プラン可）、Backlog、Jira | 0~3,000円 |
| **ソースコード管理** | GitHub（Pro: $4/月）、GitLab | 0~500円 |
| **開発環境** | VS Code（無料）、JetBrains（年額35,000円~） | 0~3,000円 |
| **クラウド** | AWS / GCP / Azure（個人利用枠あり） | 0~5,000円 |
| **デザイン** | Figma（無料プラン可）、Canva | 0~1,500円 |
| **ファイル管理** | Google Workspace（月680円~）、Dropbox | 680~1,500円 |
| **セキュリティ** | 1Password / Bitwarden、VPN | 0~1,000円 |

- **ポイント**: SaaSの年間契約は割引があるが、初年度は月額契約で試してから年額に切り替えるのが安全。すべてのSaaS費用は「通信費」または「消耗品費」として経費計上できる。

#### 11-4. セキュリティ対策
- **手順**: フリーランスは自分でセキュリティを管理する必要がある。
  - PC のパスワード設定・暗号化（BitLocker / FileVault）
  - パスワードマネージャーの導入
  - VPN の利用（公共Wi-Fi使用時）
  - 定期的なバックアップ（3-2-1ルール: 3コピー、2種類のメディア、1つはオフサイト）
  - ウイルス対策ソフト
- **ポイント**: クライアントから「ISMS対応」「セキュリティチェックシート」の記入を求められることがある。基本的なセキュリティ対策は必ず実施しておく。

### つまづきポイント
- **按分計算ミス**: 家賃の按分比率を過大にして税務調査で指摘される。面積比は客観的に計測し、写真付きで記録しておく。
- **SaaSの契約管理忘れ**: 使わなくなったSaaSの月額課金が続いている。四半期ごとにサブスクリプションを棚卸しする。
- **セキュリティインシデント**: クライアントのデータを漏洩してしまう。損害賠償問題に発展するリスクがある。

### コスト・期間
| 項目 | 目安 |
|------|------|
| 自宅（家事按分経費） | 月 15,000~30,000円（家賃により異なる） |
| コワーキングスペース | 月 5,000~30,000円 |
| SaaSツール合計 | 月 3,000~15,000円 |
| PC・周辺機器 | 15~30万円（初期投資） |
| 所要期間 | 環境構築 1~3日 |

## 12. 法人化の判断基準

> **WHY THIS MATTERS**: 法人化すれば節税できると聞くが、時期を間違えると逆にコストが増える。所得が一定水準を超えてからの法人化が鉄則。

### タスク一覧

#### 12-1. 法人化を検討すべきタイミング
- **手順**: 以下の条件に複数当てはまる場合、法人化を検討する。
  - **所得（売上 - 経費）が800~1,000万円を超えた**: 個人の所得税+住民税（最大55%）が、法人税（約23~30%）+ 役員報酬の所得税を上回るライン
  - **売上が1,000万円を超えた**: 消費税の課税事業者になるが、法人設立で最大2年間の免税期間を得られる可能性がある（ただしインボイス登録している場合は免税にならない）
  - **取引先から法人化を求められた**: 大企業は個人事業主との取引を避ける場合がある
  - **従業員を雇いたい**: 社会保険加入の義務（法人は社長一人でも社会保険加入義務あり）
  - **信用力を高めたい**: 法人は登記情報が公開されるため、社会的信用が高い
- **ポイント**: 「売上」ではなく「所得（利益）」で判断すること。売上1,500万円でも経費が多ければ所得は低い場合がある。

#### 12-2. 個人事業主と法人の比較
- **手順**: 以下の比較表で判断材料を整理する。

| 項目 | 個人事業主 | 法人（合同会社 / 株式会社） |
|------|-----------|-------------------------|
| 設立費用 | 0円 | 合同会社 6万円~ / 株式会社 25万円~ |
| 税率 | 所得税 5~45% + 住民税10% | 法人税 約15~23% + 法人住民税 |
| 赤字の繰越 | 3年 | 10年 |
| 社会保険 | 国保 + 国民年金 | 健康保険 + 厚生年金（会社と折半） |
| 決算 | 確定申告（個人で可能） | 法人税申告（税理士ほぼ必須） |
| 経費の幅 | やや狭い | 広い（役員報酬、社宅、生命保険等） |
| 社会的信用 | やや低い | 高い |
| ランニングコスト | 低い | 高い（法人住民税の均等割 年7万円~、税理士費用等） |

#### 12-3. 合同会社 vs 株式会社
- **手順**: 法人化する場合、どちらの法人形態にするか選択する。

| 項目 | 合同会社（LLC） | 株式会社 |
|------|----------------|---------|
| 設立費用 | 約6万円（登録免許税のみ） | 約25万円（登録免許税20万円 + 定款認証5万円） |
| 定款認証 | 不要 | 必要（公証役場で5万円） |
| 意思決定 | 出資者全員の合意 | 株主総会 + 取締役会 |
| 知名度 | まだ低い | 高い |
| 役員任期 | なし | 最長10年（登記変更が必要） |
| 上場 | 不可 | 可能 |
| おすすめ | 1人で事業を行うフリーランス | 将来の資金調達・拡大を見据える場合 |

- **ポイント**: **1人でIT/コンサル事業を行う場合は合同会社が合理的**。設立費用が安く、ランニングコストも低い。「株式会社」の方が信用力は高いが、フリーランスの法人化では合同会社で十分なケースが多い。Amazonジャパンや Apple Japan も合同会社。

#### 12-4. 法人化のシミュレーション
- **手順**: 法人化した場合の税負担をシミュレーションする。
  - **個人事業の場合**（所得1,000万円の例）:
    - 所得税: 約176万円（所得税率33%、控除額1,536,000円）
    - 住民税: 約100万円（10%）
    - 事業税: 約35万円（5%、290万円控除後）
    - 合計: 約311万円
  - **法人の場合**（売上同額、役員報酬600万円の例）:
    - 法人税等（法人利益400万円に対して）: 約90万円
    - 役員報酬の所得税+住民税: 約90万円
    - 社会保険料（会社負担分）: 約90万円
    - 合計: 約270万円 + ランニングコスト（税理士費用、法人住民税均等割等）
  - **差額**: 約40万円の節税 - ランニングコスト = 実質メリットは所得次第
- **ポイント**: 必ず税理士にシミュレーションを依頼すること。所得の水準によっては法人化のメリットがほとんどない場合もある。

#### 12-5. 法人化の手続き（概要）
- **手順**: 法人化を決定した場合の大まかな流れ。
  1. 定款の作成（電子定款がおすすめ。紙の場合は印紙代4万円）
  2. （株式会社の場合）公証役場で定款認証
  3. 法務局で設立登記
  4. 税務署に「法人設立届出書」等を提出
  5. 年金事務所で社会保険の手続き
  6. 個人事業の廃業届を提出
  - 全体の所要期間: 2~4週間
  - 司法書士に依頼する場合の費用: 5~15万円
  - freee会社設立やマネーフォワード会社設立を使えば自分でも可能（登録免許税以外は無料）
- **ポイント**: 法人化のタイミングは「個人事業の確定申告後」が手続き上スムーズ（年の途中だと個人と法人の両方で確定申告が必要になる）。

### つまづきポイント
- **早すぎる法人化**: 所得が低い段階で法人化すると、法人住民税の均等割（年間約7万円）や税理士費用（年間30~50万円）だけが負担になる。最低でも所得800万円を安定的に超えてから検討する。
- **消費税の免税メリットの過大評価**: インボイス登録をしている場合、法人化しても消費税の免税にはならない。2023年以前の「法人化で2年間免税」のメリットは大幅に縮小している。
- **社会保険料の見落とし**: 法人化すると社長1人でも社会保険加入義務がある。会社負担分を含めると保険料が増えるケースもある（特に役員報酬が高い場合）。

### コスト・期間
| 項目 | 目安 |
|------|------|
| 合同会社の設立費用 | 約6万円（電子定款 + 登録免許税） |
| 株式会社の設立費用 | 約25万円（定款認証5万円 + 登録免許税20万円） |
| 司法書士への依頼 | 5~15万円 |
| 法人住民税（均等割） | 年間 約7万円（赤字でも発生） |

| 所要期間 | 2~4週間（登記完了まで） |
